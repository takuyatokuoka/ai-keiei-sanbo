# 事業検証レポート：MeetMemo（AI議事録・タスク通知SaaS）

**作成日：** 2026-06-12  
**対象事業：** 会議録音をAIが自動で議事録化し、タスクを担当者に通知するSaaS「MeetMemo」

---

## 1. 課題仮説の検証

### この課題は実際に多くの人が抱えているか

**結論：YES。議事録作成の負担とタスク管理の抜け漏れは、ビジネスパーソン全体が抱える定量的に証明された課題である。**

キヤノンマーケティングジャパンが全国のビジネスパーソン（20〜59歳）を対象に実施した調査によると：

- **67%** が議事録・発言録の作成に「負担を感じている」
- 1週間あたりの平均作成時間は **6.13時間**（年換算で **約320時間**）
- 20代に限ると週 **8.46時間** と特に負担が大きい
- **70%** がAIサポートツールの利用を「希望している」
- 一方、実際にAI・DXが導入されている現場は **わずか1.4%**

希望と現実の間に約68ポイントの「導入ギャップ」が存在しており、強い潜在需要が確認できる。

### 課題を裏付けるデータと事例

| 課題 | 内容 |
|------|------|
| **議事録作成の時間コスト** | 年間320時間は、年収600万円の社員換算で約90万円分の人件費に相当する |
| **タスク管理の抜け漏れ** | 口頭で決まった「誰が・何を・いつまでに」が議事録に残らないまま会議が終わるケースが常態化 |
| **会議コストの膨張** | パーソル総合研究所の試算では「無駄な会議」による企業損失は年間15億円規模 |
| **リモートワーク後の課題深刻化** | オンライン会議の増加で録音・録画は容易になったが、文字起こし・要点整理・タスク化の工数は変わっていない |

> **出典：** [キヤノンMJ「ビジネスパーソンは年間320時間を議事録作成に費やしている」調査](https://prtimes.jp/main/html/rd/p/000000939.000013943.html)  
> **出典：** [パーソル総合研究所「ムダな会議による企業の損失は年間15億円」](https://rc.persol-group.co.jp/thinktank/column/201812130003.html)

## 2. 市場規模の推定

### 国内市場規模（TAM・SAM・SOM）

#### TAM（獲得可能な最大市場）
国内のAI議事録・会議支援ツール市場全体。

- **グローバル市場（2024年）：** 27.8億ドル（約4,200億円）
- **グローバル市場（2034年予測）：** 272.9億ドル（約4.1兆円）、**CAGR 25.62%**
- **国内推定（2025年）：** グローバルの5〜6%として約 **200〜250億円**
- **国内AI商談分析・議事録作成サービス成長率：** **52%/年** と予測

#### SAM（サービス提供可能な市場）
国内の中小〜中堅企業（従業員10〜500名）を対象とした市場。日本の就業者数は約6,700万人であり、会議参加者のうち5〜10%が能動的に使うツールとしても300〜700万ユーザーが潜在市場となる。

- **推定規模（2026年）：** 約 **60〜100億円**

#### SOM（現実的な獲得可能市場）
初期3年間でのターゲット：Slack・Teams利用企業の中でタスク管理ツールと連携ニーズの高い中小企業（50名以下）。

- **推定規模（3年後）：** 約 **5〜15億円**（ARR換算）

### 海外市場規模

| 指標 | 数値 |
|------|------|
| 2024年グローバル市場 | 27.8億ドル（約4,200億円） |
| 2034年予測 | 272.9億ドル（約4.1兆円） |
| CAGR | 25.62% |

> **出典：** [Landbridge AI「278億ドル市場が証明！企業の78%が導入するAI議事録」](https://www.landbridge.ai/columns/c7595e13-f638-4cf2-8052-49452f99a030)

### 市場の成長トレンド

以下の要因が市場を急拡大させている。

1. **生成AI（LLM）の汎用化：** GPT-4o・Geminiなどの高精度モデルが低コストでAPI提供され、文字起こし→要約→タスク抽出のパイプライン構築が容易になった
2. **リモート・ハイブリッドワークの定着：** Zoom・Teams・Google Meetが会議インフラとして常用されており、録音データの取得が前提となった
3. **インボイス制度・コンプライアンス強化：** 会議内容の記録・保管義務が高まり、議事録のデジタル化需要が増大
4. **タスク管理ツール（Notion・Asana・Jira）普及：** 会議で決まったタスクをツールに自動連携する「最後の1マイル」の自動化ニーズが顕在化

## 3. 主要競合3社の調査

### 競合1：Notta（Notta Inc.）

| 項目 | 内容 |
|------|------|
| **サービス概要** | 日本語文字起こし精度98.7%を誇るAI議事録SaaS。58言語対応。Zoom・Teams・Google Meet完全対応 |
| **料金** | 個人プラン：2,200円/月、チームプラン：8,800円/月〜 |
| **強み** | 日本語精度が業界最高水準。GPT-4ベースの要点自動抽出。モバイルアプリが充実 |
| **弱み** | 無料プランは録音3分まで（実質課金前提）。タスク抽出〜担当者への自動通知機能がない。海外発サービスのため国内データセンター保証が不明確。TrustPilotで課金トラブルの報告多数 |

### 競合2：tl;dv（tl;dv GmbH）

| 項目 | 内容 |
|------|------|
| **サービス概要** | 無料から使える海外発AI議事録ツール。無制限の録画・文字起こしが無料枠あり |
| **料金** | 無料プランあり、有料プランは月額$10〜（年間契約時） |
| **強み** | 無料枠が広く導入ハードルが低い。コストパフォーマンスが高い |
| **弱み** | **Microsoft Teams非対応**（日本企業の主要ツールが使えない）。データが海外サーバー保存のため機密会議には不向き。タスク通知機能なし。日本語サポートが手薄 |

### 競合3：Rimo Voice（Rimo合同会社）

| 項目 | 内容 |

| **料金** | 要問い合わせ（エンタープライズ価格帯） |
| **強み** | 国内サーバー・日本語サポート・セキュリティ対応が手厚い。Slack共有機能あり。2026年にタスク自動抽出「Rimo Actions」をリリース |
| **弱み** | タスク抽出はアドオン機能として後付けで追加された設計で、担当者への自動通知・進捗管理までは未対応。中小企業には価格が高く、導入ハードルがある |

## 4. 差別化ポイントの整理

競合分析から見えた構造的な「空白地帯」は、**「議事録作成」と「タスク管理・通知」を一気通貫でつなぐプロダクトの不在**である。既存サービスはいずれも「文字起こし・要約」で完結しており、「誰が・何を・いつまでにやるか」を担当者に自動で届けるところまで設計されていない。

### 差別化軸1：「会議→アクション」の完全自動化パイプライン

- 既存競合はすべて「議事録を作って終わり」。タスクの担当者割り当てと通知は人間が手動で行う
- MeetMemoは録音アップロード/録画連携→AI要約→タスク自動抽出→担当者へSlack/Teams/メール通知→期日リマインダーまでを自動化する
- これにより「会議後の議事録送付・タスク割り振り作業」を**ゼロ**にする価値提案が可能

### 差別化軸2：Teams完全対応 × 国内データセンターで法人信頼を獲得

- tl;dvはTeams非対応という致命的な欠陥があり、日本企業の主力コミュニケーションツールを切り捨てている
- Nottaは海外サービスのためデータ保管場所が不透明で、機密会議・個人情報が絡む会議への導入に社内承認が下りにくい
- MeetMemoはZoom・Teams・Google Meet全対応 ＋ 国内データセンター保管を標準仕様とし、情報セキュリティポリシーの厳しい中小企業にも導入可能な状態を最初から担保する

### 差別化軸3：タスク管理ツールとのネイティブ連携でワークフローに溶け込む設計

- Notion・Asana・Jira・Backlogなどのタスク管理ツールにタスクを自動登録する連携機能を最初から搭載
- 既存競合はSlack共有止まりだが、MeetMemoはプロジェクト管理ツール上に「誰が・何を・いつまでに」を直接生成するため、チームが別途転記する工数がなくなる
- 中小企業の非エンジニア職（営業・総務・管理職）でも使えるノーコード設定UIを実現することで、ITリテラシーを問わない横断導入が可能になる
