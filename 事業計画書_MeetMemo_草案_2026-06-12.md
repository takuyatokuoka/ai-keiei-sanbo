## 1. エグゼクティブサマリー

### サービス概要

**MeetMemo** は、会議の録音をAIが自動で議事録化し、タスクを抽出して担当者へ通知する、従業員50名以下の中小企業向けSaaSである。

ビジネスパーソンは年間平均**319.6時間**を議事録作成に費やしているにも関わらず、AI化が進んでいる現場はわずか**1.4%**（キヤノンMJ調査）。さらに、既存の競合サービスはいずれも「文字起こし・要約」で完結しており、「タスクを担当者に届ける」最後の1マイルが未解決のまま放置されている。

MeetMemoは「会議が終わった瞬間、タスクが担当者に届く」という一気通貫の自動化で、このギャップを解消する。

### 事業ハイライト

| 項目 | 内容 |
|------|------|
| **価格** | 月額 3,000円 / ユーザー |
| **ターゲット** | 従業員50名以下の中小企業（日本国内約358万社） |
| **ARR目標** | 1年目：5,000万円 ／ 3年目：2.5億円 |
| **差別化** | タスク抽出→担当者自動通知まで一気通貫（競合未対応） |
| **参入障壁** | 国内データセンター × Teams/Zoom/Google Meet全対応 × IT導入補助金対象 |

## 2. 課題と解決策

### 課題：議事録・タスク管理の「二重苦」

中小企業の会議には、構造的に解決されていない2つの課題がある。

#### 課題①：議事録作成の膨大な工数

| データ | 出典 |
|--------|------|
| 年間平均319.6時間を議事録作成に費やす | キヤノンMJ調査 |
| 67%が議事録作成に「負担を感じる」 | キヤノンMJ調査 |
| AIツール利用を「希望する」70%に対し、導入済みはわずか1.4% | キヤノンMJ調査 |
| 20代は週8.46時間を議事録に費やす（若手への集中負担） | キヤノンMJ調査 |

年収600万円の社員が年間320時間を議事録に費やすと、人件費換算で**約92万円/人**の損失になる。10名規模の企業なら年間数百万円規模のコストが「見えない形」で垂れ流されている。

#### 課題②：タスクの抜け漏れと追跡コスト

- 会議で「誰が・何を・いつまでにやるか」が決まっても、担当者への共有は人間が手動で行うのが現状
- リモート・ハイブリッドワーク（定着率65%）では口頭の温度感が伝わらず、タスク漏れがさらに深刻化
- 「パーソル総合研究所」調査では、ムダな会議（意思決定されない会議・フォローアップされない会議）による企業損失は年間**15億円**

---

### 解決策：録音から通知まで、全自動パイプライン

```
会議録音（Zoom / Teams / Google Meet）
        ↓  自動取得
   AI文字起こし（Whisper API、日本語精度98%超）
        ↓  LLM処理
   議事録生成 ＋ タスク抽出（担当者名・期日・内容）
        ↓  自動配信
   担当者へ通知（Slack / Teams / メール）
        ↓  タスク登録
   Notion / Asana / Backlog に自動登録
```

**従来の作業時間：会議後 60〜90分 → MeetMemo導入後：0分**

## 3. 市場規模（TAM・SAM・SOM）

### TAM（獲得可能な最大市場）

**国内のAI議事録・会議支援ツール市場全体**

- **2022年実績：** 409億円（前年比125.5%成長）
- **2023年実績：** 516億円
- **2027年予測：** **2,880億円**（年平均成長率25.3%）
- **グローバル市場CAGR：** 25.62%（2024〜2034年）
- **AI商談分析・議事録作成サービスの国内成長率：** **52%/年**

> **出典：** デロイト トーマツ ミック経済研究所「クラウド型経費精算システム市場の実態と展望 2024年度版」

---

### SAM（サービス提供可能な市場）

**国内の中小企業（従業員50名以下）向け市場**

- 日本の従業員50名以下の企業数：約**358万社**（中小企業庁）
- うちZoom/Teams利用 × 月10回以上の定期会議：推定**30万社**
- 平均ユーザー数10名 × 月額3,000円 × 12ヶ月 = **SAM推定：1,080億円**

---

### SOM（現実的な獲得可能市場）

**初期3年間のターゲット獲得シェア**

- 初年度目標：500社（SAMの0.17%）
- 3年目目標：2,000社（SAMの0.67%）
- **3年目ARR：約2.4〜3億円**

市場の99%以上がまだ未開拓であり、SOM達成は保守的な目標設定である。

## 5. 競合優位性

### 競合比較マトリクス

| 機能・特性 | **MeetMemo** | Notta | Otter.ai | LINE WORKS AiNote |
|-----------|:---:|:---:|:---:|:---:|
| AI文字起こし | ✓ | ✓ | ✓ | ✓ |
| AI要約・議事録生成 | ✓ | ✓ | ✓ | ✓ |
| タスク自動抽出 | ✓ | ✗ | ✗ | △（2026年4月追加、基礎的） |
| **担当者への自動通知** | **✓（中核機能）** | **✗** | **✗** | **✗** |
| Notion/Asana自動登録 | ✓ | ✗ | ✗ | ✗ |
| Teams完全対応 | ✓ | ✓ | ✓ | ✓ |
| 国内データセンター | ✓ | △不明 | ✗（米国） | ✓ |
| 中小企業向け低価格 | ✓ | ✓ | ✓ | △（チームプラン19,800円/月） |
| IT導入補助金対象 | ✓（申請予定） | ✗ | ✗ | ✓ |

### 各競合の致命的な弱点

**Notta：**
- タスク通知機能なし。文字起こしで完結する設計
- 無料プランは3分のみで実質有料前提
- 海外発サービスのためデータ保管場所が不透明（法人セキュリティ審査の障壁）

**Otter.ai：**
- 米国発で日本語サポートが薄く、敬語・業界用語の精度に課題
- タスク通知機能なし
- 日本のIT導入補助金制度に非対応

**LINE WORKS AiNote：**
- 2026年4月にタスク抽出機能を追加したが「一覧表示」にとどまり、担当者への自動通知・外部ツール連携は未対応
- チームプランが月額19,800円（100時間枠）と価格体系がシートではなく時間ベース。ユーザー課金型のMeetMemoと比較して使い方によって割高になる
- LINE WORKS本体との連携が前提であり、Slack中心の企業には導入しにくい

### MeetMemoが勝てる3つの軸

1. **「通知まで届ける」唯一性：** 会議→議事録→タスク→通知の一気通貫は現時点で競合ゼロ
2. **セキュリティと使いやすさの両立：** 国内サーバー ＋ シンプルUXで「IT担当不在の中小企業」でも導入できる
3. **エコシステム連携の深さ：** Notion/Asana/Backlog/Jiraへのタスク自動登録で、既存の仕事の仕方に自然に溶け込む

## 6. 実行計画・ロードマップ（最初の1年）

### フェーズ概要

```
Phase 1          Phase 2            Phase 3             Phase 4
[開発・準備]    [ベータ・検証]    [正式ローンチ]      [成長加速]
Month 1-3   →   Month 4-6    →    Month 7-9    →    Month 10-12
```

---

### Phase 1：開発・準備（Month 1〜3）

**目標：** プロダクトのコア機能を完成させ、50社のベータ候補を確保する

| アクション | 内容 |
|-----------|------|
| **コア機能開発** | Zoom/Teams/Google Meet録音取得 → Whisper API文字起こし → GPT-4oタスク抽出 → Slack/メール通知のパイプライン構築 |
| **インフラ整備** | 国内データセンター（AWS東京リージョン）でのデータ保管設計。セキュリティポリシー策定 |
| **ベータ候補獲得** | 士業（社労士・税理士）経由で50社のベータ参加企業を確保。LinkedIn/SNSでの認知形成開始 |
| **補助金申請準備** | IT導入補助金（デジタル化・AI導入補助金2026）の登録ベンダー申請手続き開始 |

**マイルストーン：** 社内テスト完了 ＋ ベータ申込み50社確保

---

### Phase 2：ベータ・検証（Month 4〜6）

**目標：** 50社で実使用データを収集し、製品精度とUXを磨き込む

| アクション | 内容 |
|-----------|------|
| **50社ベータ提供（無料）** | 2〜3ヶ月の無料利用期間を提供。週次でフィードバックインタビューを実施 |
| **精度改善** | 日本語議事録のタスク抽出精度をユーザーフィードバックで改善。業種別テンプレート開発 |
| **NPS計測** | Net Promoter Score（NPS）を月次計測。NPS 40以上を有料化開始の判断基準とする |
| **Notion/Asana連携** | タスク管理ツールへの自動登録機能を実装 |
| **Boxil・ITトレンド掲載準備** | 比較サイトへの掲載資料・ケーススタディ作成 |

**マイルストーン：** NPS ≥ 40 ＋ ケーススタディ10件以上

---

### Phase 3：正式ローンチ（Month 7〜9）

**目標：** 有料転換50社、MRR 150万円達成

| アクション | 内容 |
|-----------|------|
| **有料プランリリース** | ベータ参加企業に優待価格（初年度10%割引）での転換オファー |
| **比較サイト掲載** | Boxil・ITトレンドに掲載開始。レビュー獲得キャンペーンを実施 |
| **士業パートナー本格稼働** | 社労士・税理士パートナー経由の紹介フロー確立。紹介フィー（初月MRRの30%）設定 |
| **IT導入補助金活用訴求** | 補助金登録完了後、「実質半額以下で導入可能」として営業資料を全面リニューアル |
| **コンテンツSEO開始** | 「議事録 自動化」「会議 タスク 管理」「AI議事録 中小企業」などのターゲットKWで記事投稿開始 |

**マイルストーン：** 有料顧客50社 ＋ MRR 150万円

---

### Phase 4：成長加速（Month 10〜12）

**目標：** 有料顧客150社、ARR 5,000万円ペース到達

| アクション | 内容 |
|-----------|------|
| **PLGループ強化** | 無料トライアル → 有料転換フローの最適化。トライアル中のアクティベーション率70%以上を目標 |
| **Backlog・Jira連携追加** | エンジニア系企業向けに開発ツールとの連携を追加してICP（理想顧客プロファイル）を拡大 |
| **CSチーム立ち上げ** | カスタマーサクセス担当を1名採用。チャーンレートを1.5%以下に維持する |
| **メディア露出** | スタートアップメディア（THE BRIDGE・Coral Capital等）へのプレスリリース投下 |
| **シリーズA調達準備** | KPIトラッキング整備・投資家向けデータルーム構築 |

**マイルストーン：** 有料顧客150社 ＋ 月次チャーン1.5%以下 ＋ MRR 400万円
